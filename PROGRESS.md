## 已知问题 / 技术债
- 【P2】Streamlit UI 已可用，但还没有批量导入、报告导出、登录权限等完整产品化能力。
- 【P2】自动化测试还不是标准测试框架，且部分脚本依赖 `DEEPSEEK_API_KEY` 和网络。影响：CI 暂时不好接。
- 【P2】批量样例脚本会调用 DeepSeek；虽然会跳过已存在题目，但首次运行会产生 API 调用成本。影响：需要运行前提醒。
- 【P2】Chroma 当前固定为本地 `PersistentClient` 和 `chromadb==1.5.9`；未来如果要升级依赖或改成 server，必须先重新做兼容性和安全审查。
- 【P2】推荐结果目前依赖 embedding 语义相似度和轻量“解题思路标签”，还没有人工标注的相似题评测集。影响：推荐质量需要后续用更多样例调优。
- 【P3】项目文件都在根目录，后续可以按 `src/`、`tests/`、`scripts/` 整理，但现在先保持简单。

## 重要决策记录
- 【2026-06-13】决定用 SQLite 保存知识点和错题，因为当前是单机本地助手，SQLite 足够轻量，且外键能保证错题与知识点关联一致。
- 【2026-06-13】决定 DeepSeek API key 只从 `DEEPSEEK_API_KEY` 环境变量读取，代码和文档不写真实 key。
- 【2026-06-13】决定 Chroma 只用本地 `PersistentClient`，不启动 HTTP server，不开放网络端口。
- 【2026-06-13】决定相似题推荐做两层：同知识点用数据库查，跨知识点用向量检索查“思路相近”。
- 【2026-06-13】决定运行时数据库和向量库不提交 git，靠脚本重建测试数据。

## 给下一个对话的备注
