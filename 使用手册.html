1. 这个工具是干嘛的

它不是普通题库，也不是单纯的聊天机器人。它更像一个“错题学习工作台”：你把学生真实做错的题录进去，它帮你判断知识点、生成讲解、整理错题本、看薄弱点，并推荐下一步适合练的相似题。

录入真实错题
自动判断知识点
生成对症讲解
沉淀错题本
推荐相似题
最重要的一句话 想测它有没有学习价值，要用真实错题测。模拟错题只能证明功能跑通，真实错题才能看出讲解和推荐是否真的帮到孩子。

2. 如何启动网页

打开 PowerShell，运行下面命令：

cd D:\k12_math_mistake_helper
python -m streamlit run app.py

然后在浏览器打开：

http://localhost:8501
DeepSeek API key 如果要用“自动归类、生成讲解、AI 学情报告”，必须先在系统环境变量里配置 DEEPSEEK_API_KEY。如果没有配置，网页仍能看错题本，但 AI 按钮会不可用或生成失败。

3. 第一次应该怎么测

建议不要一上来乱点。按这个顺序测，最容易理解它的价值。

先录入一道真实错题。
从孩子最近做错的题里挑一道，最好是能写清楚错因的。
让系统自动归类。
保存后看它把题归到哪个知识点，比如“一元二次方程的解法”。
去“错题讲解”生成讲解。
重点看它有没有说清楚“学生到底错在哪一步”。
去“相似题推荐”看下一题。
看同知识点和思路相近的题是否合理。
录入 3-5 道后看“学情分析”。
错题太少时，学情报告参考意义不大。

4. 概览首页（打开就看到的第一页）

启动网页后，默认停在“概览”页。它把最关键的信息集中在一屏，先看这里，再决定去做什么。

区块显示什么怎么用
四个指标卡有效错题、掌握率、待复习、已掌握一眼看整体进度，掌握率会用强调色突出
快捷操作录入新错题 / 去复习 / 查看学情分析点一下直接跳到对应页面，不用再找侧栏
薄弱知识点 Top 5错题集中的具体知识点，带掌握率进度条决定先复习哪个知识点
掌握状态分布 / 薄弱大方向未掌握·复习中·已掌握的数量，以及按大方向聚合看结构性薄弱在哪个方向
最近录入最新几道错题的卡片快速回看刚录入的题，点“查看全部错题”进错题本
还没有错题时 概览页会显示一个“欢迎”提示和一个“录入第一道错题”按钮，点它就能直接开始。

5. 侧栏导航与工具

网页左侧是导航栏：所有页面都从这里切换（概览、录入错题、错题讲解、错题本、学情分析、相似题推荐）。导航下面还有常用数据和工具。

区域是什么意思怎么看
导航六个页面的入口，点哪个就进哪个当前页会高亮；也可以在概览页用快捷按钮跳转
有效错题 / 待复习去重后的错题数和待复习数随时掌握数据规模
同步错题向量把 SQLite 错题同步到 Chroma做相似题推荐前可以点一下
演示数据（折叠）检查 48 道模拟错题是否已录入真实使用时不一定要点
环境信息（折叠）DeepSeek、SQLite、Chroma 状态DeepSeek 显示“已配置”才能用 AI 功能
“原始记录”和“有效错题”为什么不一样 因为测试时可能重复录入同一道题。统计会把“同题、同错答、同知识点”的重复记录合并，只保留最新一条，所以有效错题可能少一些。此外，现在录入前会自动查重，重复的题会被拦下，不再写进库。

6. “录入错题”怎么用

这是最重要的入口。真实使用时，每一道错题都从这里开始。

需要填写什么

字段怎么填例子
题目内容把题目完整写进去解方程：x^2 - 7x + 12 = 0。
学生错误答案学生当时写错的答案x = 3
学生错误原因能写就写，越具体越好因式分解后只写了一个根，漏掉另一个解
正确答案标准答案x = 3 或 x = 4
详细解析老师解析或参考答案过程(x - 3)(x - 4) = 0，所以 x = 3 或 x = 4
年级 / 题型 / 难度按实际情况选择初三 / 解答 / 基础

自动归类 vs 手动选择知识点

DeepSeek 自动归类 适合真实使用。系统会判断题目属于哪个初中数学知识点。如果题目是小学、高中或非数学题，会提示超范围。
手动选择知识点 适合你已经知道知识点，或者没有配置 DeepSeek API key 时使用。它不会调用 AI 自动判断。

保存后生成讲解

勾选后，保存错题之后会立刻调用 DeepSeek 生成讲解。建议第一次测试可以先不勾选，保存成功后再去“错题讲解”页面单独生成，这样更清楚每一步发生了什么。

不要只填题目 这个工具的价值来自“题目 + 学生错误答案 + 错误原因”。如果只填题目，它就只能泛泛讲题，没法真正对症。
重复录入会被拦下 如果同一个知识点下已经有“题目相同、学生错答相同”的题，保存时会提示“这道题已存在（#编号）”，不会重复写库。想改内容请去错题本里“编辑”，不要重复录入。

7. “错题讲解”怎么用

这里用来给某一道错题生成讲解，讲解会围绕学生这次具体错误展开。

在下拉框选择一道错题。
检查题目、学生答案、错因是否正确。
点击“生成 / 查看讲解”。
如果已经生成过，系统会优先读取缓存，不会重复调用 DeepSeek。
如果你改过题目信息或想重新生成，可以勾选“重新生成”。

生成的讲解通常包含四个部分：

你错在哪里 一起想一想 正确解法 举一反三

改过题目后，旧讲解会自动作废 如果你在错题本里编辑了题目或答案，这道题原来缓存的讲解会被清空，下次进这里会重新生成，保证讲解和最新内容一致。

8. “错题本”怎么用

这里是复习中心。不是为了看所有题，而是为了知道今天该复习哪几题。

视图作用适合什么时候用
今日复习按遗忘曲线算出“今天到期该复习”的题每天打开后最优先看这个
待复习显示所有未掌握和复习中的题（不看到期与否）想把欠账整体过一遍时用
全部显示所有错题想整体翻看时用
按知识点只看某个知识点的错题专题复习时用
按状态按未掌握、复习中、已掌握筛选整理复习进度时用
按年级只看初一、初二、初三题目按学段复习时用

“今日复习”是怎么排的

这是间隔复习（遗忘曲线）：一道题复习得越多次，下次回来的间隔越长（约 1 → 3 → 7 → 15 → 30 天）。“未掌握”的题最多隔 1 天就回来，“已掌握”的题只做长间隔巩固。你在“管理”里更新掌握状态并勾选“计入复习次数”后，这道题会自动进入下一个复习间隔。排序上，未掌握优先于复习中，逾期越久越靠前。

搜索与翻页

视图下方有一个搜索框，输入关键词即可在当前视图里过滤（匹配题目、知识点或学生答案）。题多的时候用分页：可以选每页显示 5 / 10 / 20 / 50 道，再用页码跳转，不会一次把所有题堆在一屏。

掌握状态怎么改

展开每道题下面的“管理 #编号”，在里面更新掌握状态；保存后页面会立即刷新显示新状态。

  • 未掌握：现在还不会，或者看讲解后仍不稳。
  • 复习中：这次看懂了，但还需要再做类似题确认。
  • 已掌握：能独立做对同类题，可以暂时退出待复习。

编辑与删除

编辑内容 在“管理”面板里可以直接改题目、学生答案、错因、正确答案、详细解析、年级、难度，保存即可。改过内容后旧的 AI 讲解会自动清空。
删除错题 勾选“我确认删除这道错题”后才能点“删除”。删除会同时清掉这道题在相似题向量库里的记录，删了就不再出现在推荐里。
建议用法 每次复习完一道题，不要立刻都改成“已掌握”。更稳的做法是先改成“复习中”，过两天再做相似题，如果还能做对，再改成“已掌握”。

9. “学情分析”怎么看

这里不是给单题讲解，而是看一段时间的整体情况。

指标意思怎么看
有效错题去重后的错题数比原始记录更适合分析
掌握率已掌握错题占有效错题比例刚开始低很正常
待复习未掌握 + 复习中的题这是当前最该处理的题
薄弱知识点错题比较集中的具体知识点用来决定先复习哪里
大方向按上级知识点聚合比如方程与不等式、函数等方向

什么时候生成 AI 学情报告

建议至少录入 3-5 道真实错题后再生成。只有 1 道题时，报告容易过度解读。

学情报告适合给谁看 它更适合家长或老师看，用来判断“孩子最近哪里薄弱、接下来怎么复习”。如果只是学生自己做一题，先看“错题讲解”和“相似题推荐”更直接。

10. “相似题推荐”怎么用

这是这个项目比较特别的地方：它不只按知识点推荐，还会按解题思路推荐。

第一层：同知识点 从数据库里找同一个知识点下的其他题。比如当前题是“一元二次方程”，就推荐其他一元二次方程题。
第二层：思路相近 用 Chroma 向量检索，从其他知识点里找思路相近的题。比如因式分解、判别式、移项变号这些共同能力。

怎么操作

选择目标错题。
设置同知识点数量和思路相近数量，默认 5 道就够。
保持“先同步向量库”勾选，尤其是刚录入新题后。
点击“生成推荐”。
先做同知识点题，再做思路相近题。

推荐结果怎么看

同知识点 表示直接巩固。思路相近 表示知识点不同，但思维方法或易错点相近。

例如：一元二次方程里因式分解漏根，可能推荐到分式运算里的因式分解题，或者二次函数里求 x 轴交点的题。

11. 一道真实错题应该怎么准备

你可以按这个模板准备，填进网页里：

题目内容：
学生错误答案：
学生错误原因：
正确答案：
详细解析：
年级：
题型：
难度：

示例

题目内容解方程：x^2 - 7x + 12 = 0。
学生错误答案x = 3
学生错误原因因式分解后只写了一个根，漏掉另一个因式对应的解。
正确答案x = 3 或 x = 4
详细解析因式分解得 (x - 3)(x - 4) = 0，所以 x = 3 或 x = 4。
年级 / 题型 / 难度初三 / 解答 / 基础
如果不知道错误原因怎么办 可以先写“学生未说明原因，请根据错误答案分析”。但是如果能写出真实错因，AI 讲解会更像老师对着这个孩子讲。

12. 常见问题

录入时提示“这道题已存在”怎么办？

说明同知识点下已经有一道题目和学生错答都相同的题，系统按查重规则拦下了。如果只是想改内容，去错题本里“编辑”那道题；如果确实是不同的题，检查一下题目或错答是不是填得完全一样。

为什么有很多模拟题？

那是为了测试相似题推荐是否能跑通。真实使用时，重点应该逐步录入孩子自己的错题。

为什么点“生成讲解”要等一会儿？

因为它会调用 DeepSeek。题目越长、网络越慢，等待越明显。

为什么第一次用相似题推荐比较慢？

因为要加载中文 embedding 模型 BAAI/bge-small-zh-v1.5，并把错题同步到本地 Chroma。后面会快一些。

为什么推荐结果有时看起来不是同一个知识点？

这是正常的。“思路相近”故意不限制同知识点，它找的是共同解题能力，比如因式分解、判别式、移项变号、数形结合。

数据库和向量库在哪里？

SQLite 数据库是 mistakes.db，Chroma 向量库是 mistake_chroma_db/。它们都是本地运行时数据，不上传到 GitHub。

能不能给真实学生长期用？

当前适合本地测试和个人使用。如果要给真实学生长期使用，需要补隐私说明、数据清空、备份、权限控制等产品化能力。
